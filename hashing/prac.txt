ARRAY HASHING VS MAP (C++)

WHAT IS HASHING?

Hashing is a technique used to store and retrieve data (usually frequency) in constant time O(1).

TYPES OF HASHING
Array Hashing

int hash[100000] = {0};
hash[x]++;

Uses index as key
Direct mapping: value → index
Map Hashing (unordered_map)

unordered_map<int, int> mp;
mp[x]++;

Uses key-value pairs
Dynamic storage
COMPARISON
Feature | Array Hashing | unordered_map

Time Complexity | O(1) | O(1) average
Worst Case | O(1) | O(n)
Memory Usage | Fixed (may waste) | Dynamic
Range Support | Limited | Unlimited
Negative Values | Not allowed | Allowed
Safety | Risk of crash | Safe
Flexibility | Low | High

WHEN TO USE ARRAY HASHING

Use when:

Values are small
Range is known

Example:
1 ≤ arr[i] ≤ 1000

int hash[1001] = {0};

WHEN TO USE unordered_map

Use when:

Values are large (like 10^9)
Values are negative
Range is unknown

unordered_map<int, int> mp;

LIMITATION OF ARRAY HASHING

Example:
arr = [1000000000]

int hash[1000000000]; → Not possible (memory error)

MAP SOLUTION

unordered_map<int, int> mp;
mp[1000000000]++;

TIME COMPLEXITY
Operation | Array | unordered_map

Insert | O(1) | O(1) average
Access | O(1) | O(1) average
Worst Case | O(1) | O(n)

MEMORY USAGE

Array:
int hash[1000000];

Allocates full memory even if unused

Map:
mp[5], mp[1000000];

Stores only required elements
ORDERED MAP (map)

map<int, int> mp;

Time: O(log n)
Stores keys in sorted order
Use when order is required
FINAL DECISION RULE

Ask:
"Can I use value directly as index?"

If YES → Use Array Hashing
If NO → Use unordered_map

EXAMPLE

arr = [1,2,1,3,2]

Output:
1 → 2
2 → 2
3 → 1

INTERVIEW TIP
Small range → Use Array (fastest)
Large range → Use Map (safe)

SUMMARY

Situation → Best Choice

Small range → Array
Large values → Map
Negative values → Map
Need sorted order → map

Golden Rule:
Array hashing is fastest, but map is safest.
